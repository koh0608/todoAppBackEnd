## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Installation

1. Create .env file: `cp .env.sample .env`
2. Update .env file values
3. Install dependencies `npm install` or `yarn`
4. `npm run start` or `yarn start`

---- 

### Nest CLI
##### Generate module
  `nest g module modules/<module-name>/<file-name> --flat`
##### Generate resolver
  `nest g resolver modules/<module-name>/<file-name> --flat`
##### Generate service
  `nest g service modules/<module-name>/<file-name> --flat`
##### Generate controller
  `nest g controller modules/<module-name>/<file-name> --flat`

---- 
### Sequelize
##### Create sequelize migration file
  `sequelize migration:create --name "<migration-file-name>"`

##### Run sequelize migration
  `sequelize db:migrate`

##### Run sequelize seeder
  `sequelize db:seed --seed <seed-file-name>`

---

### Environment (.env)
1. Before add any env to `.env` file please update the env configuration
2. Configuration file: `src/configs/env.config.ts`
---

## Notes
`.module.ts` 
: A wrapper file for (`.service.ts`, `.controller.ts`, `.resolver.ts`), imported to `app.module.ts`


`.service.ts`
: This is the file which contain all the logic for the `Rest API` or `Graphql`

`.controller.ts`
: for Rest API (GET, POST, PUT, DELETE...)

`.resolver.ts`
: for Graphql query or mutation

`.entity.ts`
: database table modeling file (table mapping)

`.dto.ts`
: defines how the data will be sent over the `graphql` or `rest api`


## Running the app

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Test

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```
